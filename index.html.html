About the Methodology

What is Agenda Process Design?

Agenda Process Design (APD) is a disciplined methodology for designing how people think together in meetings, workshops, and decision sessions. It separates what must be achieved (Task) from how people will think together to achieve it (Process).

Developed by Tony Mann and published in Facilitation — Develop Your Expertise (APMG, 2025), APD gives Facilitators a structured toolkit of named tools — each designed to perform a specific cognitive function at the right point in a meeting.

The result is meetings that produce clarity, commitment, and disciplined action — rather than wasted time, unresolved conflict, or decisions that don't stick.

This tool uses the APD bot to design your Agenda Process, then converts the output into a professional Appendix-3 formatted table — the canonical APD output format used by certified practitioners worldwide.

🔴 RED Thinking — What must be achieved

Establish the Aim, Objectives, Tasks, and Degree of Uncertainty before designing any process. RED is sacrosanct — no tools, no formats, no steps until RED is complete.

🟢 GREEN Thinking — How people will think together

Design the process using named APD tools in the correct sequence. Each tool performs a specific cognitive function — data generation, meaning stabilisation, diagnosis, evaluation, or action.

📋 Appendix-3 — The Agenda Process table

The canonical output format: Objective | Time | Task | Process | Preparation. Every step specified in sufficient detail for a Facilitator or Tech Host to run without interpretation.
